.TH JSSCLI 1 "2026-07-18" "jsscli 0.3" "Perintah Pengguna"
.
.SH NAMA
jsscli \- antarmuka baris perintah untuk API REST JSS / Frappe
.
.SH RINGKASAN
.B jsscli
[\fB\-q\fP|\fB\-v\fP...]
[\fB\-\-output\fP \fIformat\fP]
[\fB\-\-profile\fP \fInama\fP]
.I PERINTAH
[\fIpilihan\fP...]
.
.SH DESKRIPSI
.B jsscli
adalah alat bantu untuk mengelola otentikasi, akun, pengaturan,
serta operasi CRUD (buat, baca, ubah, hapus) terhadap DocType, file,
dan laporan di sistem berbasis Frappe/JSS.
Kredensial disimpan aman di keyring sistem, dan data sesi dienkripsi saat disimpan.
.
.SH PILIHAN (OPTIONS)
.SS Pilihan global
.TP
\fB\-q\fP, \fB\-\-quiet\fP
Mode senyap: tidak menampilkan pesan sukses atau petunjuk.
.TP
\fB\-v\fP, \fB\-\-verbose\fP
Menambah tingkat verbositas (bisa diulang). Mengatur log-level menjadi \fBinfo\fP
untuk \fBjsscli\fP dan \fBdebug\fP untuk \fBlibrjss\fP.
.TP
\fB\-\-output\fP \fIformat\fP
Format keluaran: \fBjson\fP, \fBpretty\fP (bawaan), \fBtable\fP,
atau \fBraw\fP.
.TP
\fB\-\-profile\fP \fInama\fP
Menggunakan profil akun tertentu alih-alih akun yang sedang aktif.
Bisa juga diatur lewat variabel lingkungan \fBJSS_PROFILE\fP.
.
.SH PERINTAH (COMMANDS)
.SS auth
Otentikasi dan manajemen sesi.
.TP
\fBjsscli auth login\fP
Masuk ke akun yang aktif dan menyimpan sesi.
.TP
\fBjsscli auth logout\fP
Keluar dari sesi (di server) dan menghapus catatan sesi lokal.
.TP
\fBjsscli auth status\fP
Menampilkan status otentikasi akun yang aktif.
.TP
\fBjsscli auth whoami\fP
Menampilkan identitas pengguna yang sedang login.
.
.SS account
Mengelola akun (profil) yang tersimpan.
.TP
\fBjsscli account add\fP \fInama\fP [\fIpilihan\fP]
Menambahkan akun baru.
.RS
.TP
\fB\-\-url\fP \fIURL\fP
URL dasar situs Frappe (wajib).
.TP
\fB\-\-mode\fP \fBsession\fP|\fBtoken\fP
Mode otentikasi (bawaan: \fBsession\fP).
.TP
\fB\-\-email\fP \fIemail\fP
Alamat email (hanya untuk mode session).
.TP
\fB\-\-password\fP \fIkata_sandi\fP
Kata sandi; jika tidak diisi akan diminta secara interaktif.
.TP
\fB\-\-api\-key\fP \fIkunci\fP
Kunci API (hanya untuk mode token).
.TP
\fB\-\-api\-secret\fP \fIrahasia\fP
Rahasia API (hanya untuk mode token).
.TP
\fB\-\-sitename\fP \fIstring\fP
Nama situs yang diharapkan untuk verifikasi.
.TP
\fB\-\-activate\fP
Langsung menjadikan akun ini sebagai akun aktif.
.RE
.TP
\fBjsscli account list\fP
Menampilkan daftar semua akun. Akun aktif ditandai dengan \fB*\fP.
.TP
\fBjsscli account use\fP \fInama\fP
Menetapkan \fInama\fP sebagai akun yang aktif.
.TP
\fBjsscli account remove\fP \fInama\fP
Menghapus akun beserta semua rahasia yang tersimpan.
.TP
\fBjsscli account show\fP [\fInama\fP]
Menampilkan detail akun (rahasia disembunyikan). Bawaan: akun aktif.
.
.SS config
Mengelola pengaturan global CLI.
.TP
\fBjsscli config set\fP \fIkunci\fP \fInilai\fP
Menyetel kunci konfigurasi. Kunci yang didukung: \fBactive_account\fP,
serta pengaturan bawaan lainnya.
.TP
\fBjsscli config get\fP \fIkunci\fP
Menampilkan nilai dari kunci konfigurasi.
.TP
\fBjsscli config list\fP
Menampilkan semua pengaturan yang ada.
.TP
\fBjsscli config init\fP
Membuat direktori dan berkas konfigurasi jika belum ada.
.TP
\fBjsscli config path\fP
Menampilkan lokasi berkas pengaturan.
.
.SS doctype
Operasi CRUD untuk DocType Frappe.
.TP
\fBjsscli doctype get\fP \fIdoctype\fP \fInama\fP
Mengambil satu dokumen berdasarkan nama.
.TP
\fBjsscli doctype list\fP \fIdoctype\fP [\fIpilihan\fP]
Menampilkan daftar dokumen dari suatu DocType.
.RS
.TP
\fB\-\-fields\fP \fIf1,f2,...\fP
Daftar field yang ingin ditampilkan, dipisahkan koma.
.TP
\fB\-\-filter\fP \fIfield,operator,nilai\fP
Menambahkan filter; bisa diulang.
.TP
\fB\-\-limit\fP \fIn\fP
Jumlah maksimum data yang diambil (bawaan: 20).
.TP
\fB\-\-start\fP \fIn\fP
Offset untuk halaman (bawaan: 0).
.TP
\fB\-\-order\-by\fP \fIfield\fP
Field untuk pengurutan.
.RE
.TP
\fBjsscli doctype create\fP \fIdoctype\fP [\fIberkas\fP]
Membuat dokumen baru dari file JSON. Jika \fIberkas\fP adalah \fB\-\fP
atau tidak diisi, akan membaca dari \fIstdin\fP.
.TP
\fBjsscli doctype update\fP \fIdoctype\fP \fInama\fP [\fIberkas\fP]
Memperbarui dokumen yang sudah ada. Membaca JSON dari \fIberkas\fP atau \fIstdin\fP.
.TP
\fBjsscli doctype delete\fP \fIdoctype\fP \fInama\fP
Menghapus dokumen. Akan meminta konfirmasi, kecuali diberi \fB\-y\fP.
.RS
.TP
\fB\-y\fP, \fB\-\-yes\fP
Lewati konfirmasi.
.RE
.
.SS file
Mengunggah dan mengunduh file.
.TP
\fBjsscli file upload\fP \fIberkas\fP \fIdoctype\fP \fInama_dok\fP \fInama_field\fP
Mengunggah file lokal dan melampirkannya ke dokumen. Maksimum ukuran file 50 MB.
.TP
\fBjsscli file download\fP \fIurl\fP [\fIpilihan\fP]
Mengunduh file dari situs Frappe.
.RS
.TP
\fB\-o\fP, \fB\-\-output\fP \fIlokasi\fP
Simpan file ke \fIlokasi\fP. Jika tidak diisi, digunakan nama dasar dari URL.
.TP
\fB\-y\fP, \fB\-\-yes\fP
Timpa file yang sudah ada tanpa bertanya.
.RE
.
.SH VARIABEL LINGKUNGAN
.TP
\fBJSS_PROFILE\fP
Nama profil akun yang akan digunakan, sama dengan \fB\-\-profile\fP.
.TP
\fBJSS_EMAIL\fP
Email bawaan saat menambah akun mode session (dipakai oleh \fBaccount add\fP).
.TP
\fBJSS_ACTIVE_ACCOUNT\fP
Menimpa pengaturan akun aktif dari \fBconfig.toml\fP.
.
.SH BERKAS (FILES)
.TP
.I ~/.config/jsscli/config.toml
Pengaturan global (akun aktif, nilai bawaan).
.TP
.I ~/.config/jsscli/accounts.toml
Data akun-akun. Rahasia tidak disimpan di sini.
.TP
.I ~/.cache/jsscli/session.enc
Catatan sesi terenkripsi.
.TP
.I ~/.cache/jsscli/.session.salt
Salt untuk derivasi kunci enkripsi sesi.
.PP
Semua data sensitif (kata sandi, API secret, kunci induk enkripsi)
disimpan di keyring sistem (misalnya gnome-keyring, Keychain di macOS).
.
.SH KODE KELUAR (EXIT STATUS)
.TP
0
Sukses.
.TP
1
Kesalahan umum.
.TP
2
Kesalahan penggunaan / input tidak valid.
.TP
3
Belum terotentikasi.
.TP
4
Kesalahan jaringan.
.TP
5
Kesalahan API.
.
.SH CONTOH
Menambah akun dan login:
.RS
.PP
\fBjsscli account add myhost --url https://erp.perusahaan.com --email admin@perusahaan.com --activate\fP
.br
\fBjsscli auth login\fP
.RE
.PP
Menampilkan daftar item dari suatu DocType:
.RS
\fBjsscli doctype list "Sales Order" --fields name,customer,status --limit 10\fP
.RE
.PP
Mengunggah file dan melampirkannya ke dokumen:
.RS
\fBjsscli file upload laporan.pdf "Sales Order" SO-00001 logo_perusahaan\fP
.RE
.PP
Mengunduh file dengan nama bawaan:
.RS
\fBjsscli file download /files/laporan.pdf\fP
.RE
.
